## Documentation Standards

### File Naming Convention
- Use UPPERCASE with underscores for documentation files
- End with `.md` extension
- Be descriptive and specific

### Content Standards
- Start with a clear title and purpose
- Include table of contents for long documents
- Add cross-references to related documents
- Include last updated date and version
- Use consistent markdown formatting

### Cross-Reference Format
```markdown
- [Document Name](DOCUMENT_NAME.md) - Brief description
```

### Update Protocol
1. When creating new documentation, add it to this index
2. Update related documents with cross-references
3. Log changes in [DEVELOPMENT_LOG.md](DEVELOPMENT_LOG.md)
4. Follow guidelines in [CLAUDE.md](CLAUDE.md)
